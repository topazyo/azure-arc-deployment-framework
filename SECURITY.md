# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in the Azure Arc Deployment Framework, please report it to us as follows:

### Contact
- **Email**: security@topazyo.com (placeholder; update with actual contact)
- **GitHub Issues**: Create a private security advisory via [GitHub Security Advisories](https://github.com/topazyo/azure-arc-deployment-framework/security/advisories/new)

Please do not report security vulnerabilities through public GitHub issues.

### Response Process
- We will acknowledge receipt within 48 hours
- We will provide a more detailed response within 7 days indicating our next steps
- We will keep you informed of our progress throughout the process

## Security Considerations
- Do not commit secrets or sensitive data to the repository
- Use secure storage for configuration files containing credentials
- Follow Azure security best practices for service principals and permissions

## Responsible Disclosure
We kindly ask that you:
- Give us reasonable time to fix the issue before public disclosure
- Avoid accessing or modifying user data
- Provide sufficient detail to reproduce the issue
